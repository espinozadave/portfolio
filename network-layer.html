Network Layer Communication Analysis

Overview

In this activity, you will analyze DNS and ICMP traffic in transit using data from a network protocol analyzer tool. You will identify which network protocol and service were impacted by the cybersecurity incident. In the Internet layer of the TCP/IP model, IP formats data packets into IP datagrams, which provide security analysts with insight into potentially suspicious traffic.

Knowing how to identify potentially malicious traffic on a network helps cybersecurity analysts assess security risks and reinforce network security. After analyzing the data, you will report your findings so that security engineers can take appropriate action.

Scope

  • Analyze packet data captured from a network protocol analyzer.
  • Identify which network protocols were involved in the incident.
  • Review DNS and ICMP traffic for irregular patterns.
  • Determine whether the activity indicated a potential network issue or security concern.
  • Document findings and prepare a follow-up report.

Key Skills Demonstrated

  • Network traffic analysis
  • Understanding the Internet layer of the TCP/IP model
  • Protocol identification (DNS and ICMP)
  • Packet inspection using tcpdump logs
  • Incident assessment and documentation

Scenario

This scenario is based on a fictional company:

You are a cybersecurity analyst working at a company that specializes in providing IT services for clients. Several customers of clients reported that they were not able to access the client company website www.yummyrecipesforme.com, and saw the error “destination port unreachable” after waiting for the page to load.

You are tasked with analyzing the situation and determining which network protocol was affected during this incident. To start, you attempt to visit the website and you also receive the error “destination port unreachable.” To troubleshoot the issue, you load your network analyzer tool, tcpdump, and attempt to load the webpage again. To load the webpage, your browser sends a query to a DNS server via the UDP protocol to retrieve the IP address for the website's domain name; this is part of the DNS protocol. Your browser then uses this IP address as the destination IP for sending an HTTPS request to the web server to display the webpage. The analyzer shows that when you send UDP packets to the DNS server, you receive ICMP packets containing the error message: “udp port 53 unreachable.”

tcpdump log

The tcpdump log shows a request from a source IP address to the DNS server attempting to resolve the IP address of yummyrecipesforme.com. The server responds with an ICMP error message stating “UDP port 53 unreachable.”

Cybersecurity Incident Report Network Traffic Analysis

Activity Summary & Findings

After reviewing the tcpdump logs, DNS queries and ICMP packets were identified within the captured network traffic. The analysis involved examining packet structure, identifying the protocols in use, and evaluating whether any traffic patterns indicated suspicious activity.

This exercise demonstrated how cybersecurity analysts can inspect network traffic to better understand communication between systems and detect unusual behavior. By analyzing packet data and documenting findings, analysts provide valuable information that allows security engineers to investigate and resolve potential network issues.

Tools Used

  • tcpdump – Captured and analyzed network packets to observe DNS requests and ICMP responses.
  • Command-Line Interface (CLI) – Used to run tcpdump commands and review packet capture logs.
  • TCP/IP Protocol Analysis – Examined DNS and ICMP behavior at the network layer to identify communication issues.

Security Recommendations

  • Ensure DNS services are properly configured and actively running on UDP port 53.
  • Implement continuous network monitoring to detect DNS service interruptions.
  • Use firewall and network rules to verify that DNS traffic is not unintentionally blocked.
  • Deploy logging and alerting systems to quickly identify DNS communication failures.
